# tests/

Unit + integration tests. Fixture creation happens once per project; committed
to git so tests are reproducible.

## Fixture requirements

Create these manually or via a one-time recording session, then commit the
files to `tests/fixtures/`:

- `mini_screen.mkv` — 30 s silent screen recording
  - 0.0–2.0 s: empty desktop (for clap flash to land)
  - 2.0–2.1 s: white flash frame (generated by `cursor-tracker/screen_flash.py`)
  - 2.1–12.0 s: type a command, watch output scroll
  - 12.0–22.0 s: static UI (no motion — dead zone)
  - 22.0–30.0 s: drag a window around
- `mini_webcam.mkv` — 30 s webcam + mic audio
  - 2.0 s: hand clap
  - 5.0–8.0 s: "Hello everyone, welcome back"
  - 10.0–15.0 s: "Umm yeah so uhh like let me show you" (fillers!)
  - 16.0–22.0 s: "Let me install this, this will take a minute" (dead-zone cue)
  - 24.0–30.0 s: "Alright that's it, see you next time"
- `mini_cursor.csv` — matching cursor trace with a `clap` row at t=2.0

## Golden outputs

Hand-generate the expected outputs from each stage and commit to
`tests/fixtures/expected_*.json`:

- `expected_sync.json` — offset + clap/flash timestamps
- `expected_words.json` — mlx-whisper output on mini_webcam
- `expected_filler_cuts.json` — 2–3 filler cut ranges
- `expected_layout_plan.json` — segments by layout
- `expected_dead_zones.json` — the 12.0–22.0 s install wait
- `expected_segments.json` — unified edit list

Some of these will change when models are updated — tests allow 10% tolerance
on filler recall and drift ± 100 ms on time boundaries.

## Running tests

```bash
source venv/bin/activate
pytest                     # all
pytest -m "not slow"       # skip LLM + render (fast unit-only)
pytest tests/test_sync.py  # single module
```
